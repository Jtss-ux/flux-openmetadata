The Problem

AI Agents Are Metadata-Blind

Enterprise data teams use OpenMetadata to catalog thousands of tables, pipelines, and dashboards. But AI agents — the tools that could help them most — have no standard way to access that context.

  • Analysts waste hours hunting for the right table manually
  • AI agents can't discover, trace lineage, or check data quality
  • No open standard bridges AI models with metadata platforms
  • OpenMetadata's rich APIs go untapped by the AI ecosystem
The Solution

OpenMetadata MCP Server

FLUX:// implements the Model Context Protocol — the open standard for AI-tool integration — exposing OpenMetadata as 6 callable tools for any compatible AI agent.

Tech Stack & Architecture

What It's Built With

🐍

MCP SERVER (Python 3.11+)

Built on the official mcp SDK. Fully async with httpx. Type-safe with Pydantic. Runs over stdio for zero-friction AI integration.

⚛️

WEB APP (React + Vite)

Retro-futuristic chat interface. Proxies directly to the OpenMetadata sandbox. Live entity count pulled on startup.

🔗

OPENMETADATA APIS USED

Search, Lineage, Data Quality, Governance/Policy, Pipelines, and Entity Details — 5 distinct API surfaces.

🤖

MCP PROTOCOL

Standard tool definitions with JSON Schema. Each tool maps 1:1 to an OpenMetadata capability. Plug in any AI agent.

MCP Tools

6 Tools — Full Catalog Coverage

search_tables Full-text search across all data assets — tables, dashboards, pipelines SEARCH API
get_table_lineage Upstream & downstream lineage with column-level granularity LINEAGE API
get_data_quality Test results, pass/fail counts, anomaly detection scores QUALITY API
get_data_asset Schema, columns, tags, owners, and usage statistics ENTITY API
list_pipelines Monitor ingestion pipeline health, status, and run history PIPELINE API
get_policy Governance rules, PII classification, access controls GOVERNANCE API

Learning & Growth

What I Learned

  • MCP is the USB-C of AI integration — one protocol, every AI client. Building a server from scratch made this crystal clear.
  • OpenMetadata's APIs are underutilized — the Search, Lineage, and Quality APIs are production-grade but rarely exposed to AI tooling.
  • Async Python is the right choice — httpx + asyncio made the server handle concurrent AI agent requests elegantly.
  • The metadata gap is real — every data team I could demo to immediately saw the value. This problem is worth solving properly.
